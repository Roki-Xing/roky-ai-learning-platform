# Review Center

## Owner

- UI: `/review`
- Server action and service: `src/server/review/actions.ts`
- Scheduling rule: `src/server/review/schedule.ts`
- Queue filter: `src/server/review/filter.ts`
- Today entry summary: `src/server/review/today-summary.ts`
- Empty state summary: `src/server/review/empty-state.ts`

## Behavior

1. `/review` only shows reviewable flashcards that are due.
2. The card answer is hidden until the user reveals it.
3. Rating options use the documented spacing:
   - `forgot`: +1 day
   - `hard`: +3 days
   - `good`: +7 days
   - `easy`: +14 days
4. Successful rating writes one `ReviewLog` and updates the flashcard counters.
5. Rating is user-scoped by `userId`.
6. `/review` shows the current queue scope and schedule summary in the sidebar.
7. Queue summary is generated by `buildReviewScheduleSummary()`:
   - default queue: `综合复习队列`
   - `source=project`: `项目卡片复习`
   - `source=code-feedback`: `代码反馈复习`
   - `source=glossary`: `术语卡片复习`
   - `source=radar`: `Radar 卡片复习`
   - `source=voice-note`: `语音笔记复习`
   - `source=thought-review`: `思路评审复习`
8. `/today` uses `buildTodayReviewSummary()` to show the lesson card count, lesson due count, global due count, and the correct CTA state.
9. Empty `/review` queues use `buildReviewEmptyState()` to show concrete next actions:
   - default queue: `/today`, `/library`, `/progress`
   - project queue: project detail, `/projects`, `/review`
   - code-feedback queue: project detail, `/review`, `/progress`

## Idempotency Boundary

- `rateFlashcard()` only updates cards where `dueAt <= now`.
- Once a card is rated, it is moved to a future `dueAt`.
- A duplicate submission for the same already-rated card returns `not_due`.
- Duplicate submissions must not create another `ReviewLog`.
- Duplicate submissions must not increment `reviewCount` or `correctCount`.
